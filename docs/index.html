Quick Start

Get roasting in under a minute. First, create an API key:

# Create your API key
                        curl -X POST https://roast-api.up.railway.app/v1/api-keys \
                        -H "Content-Type: application/json" \
                        -d '{"name": "My App", "email": "you@example.com"}'bash

Then generate your first roast:

# Generate a roast
                        curl -X POST https://roast-api.up.railway.app/v1/roast \
                        -H "Content-Type: application/json" \
                        -H "x-api-key: roast_your_key_here" \
                        -d '{
                            "theme": "GAMING",
                            "heat": 3,
                            "context": {
                            "name": "Dave",
                            "game": "Valorant",
                            "performance": "0 kills, 15 deaths"
                            }
                        }'bash

Authentication

All requests to /v1/roast require an API key. Include it in the header:

x-api-key: roast_your_key_hereheader

API keys are prefixed with roast_ for easy identification. Store your key securely — it won't be shown again after creation.

Endpoints

Base URL: https://roast-api.up.railway.app

GET /health

Health check endpoint. Returns API status and version.

POST /v1/api-keys

Create a new API key. No authentication required.

GET /v1/themes

List all available themes with descriptions and sample context fields.

POST /v1/roast

Generate a roast. Requires API key authentication.

Request Parameters

Parameter Type Required Description
theme string Yes CODING, GAMING, PRODUCTIVITY, or EDUCATIONAL
heat integer No Intensity level 1-5 (default: 2)
length integer No Maximum characters (default: 200)
context object Yes Target information including name
context.name string Yes Who to roast

Themes

Choose a theme that matches your use case. Each theme has a unique persona and roasting style.

CODING

Dev roasts for bugs, bad practices, and Stack Overflow dependency.

GAMING

Gamer burns for K/D ratios, rank struggles, and rage quits.

PRODUCTIVITY

Habit shame for broken streaks, screen time, and procrastination.

EDUCATIONAL

Student roasts for wrong answers and "I studied" excuses.

Theme-Specific Context

Theme Suggested Context Fields
CODING language, mistake, experience
GAMING game, performance, rank
PRODUCTIVITY habit, streak, screenTime
EDUCATIONAL subject, score, mistake

Heat Levels

Control the intensity of your roasts with the heat parameter.

1
Grandma Safe Gentle teasing, almost a compliment
2
Office Appropriate Light burns, no profanity (default)
3
Friend Group Solid roasts, mild profanity okay
4
Harsh But Fair Sharp wit, profanity allowed
5
Unfiltered Brutal honesty, heavy profanity

Discord.js Integration

// Discord bot slash command
async function execute(interaction) {
  await interaction.deferReply();
  
  const response = await fetch('https://roast-api.up.railway.app/v1/roast', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'x-api-key': process.env.ROAST_API_KEY
    },
    body: JSON.stringify({
      theme: 'GAMING',
      heat: 3,
      context: {
        name: interaction.options.getString('target'),
        game: interaction.options.getString('game'),
        performance: interaction.options.getString('performance')
      }
    })
  });
  
  const data = await response.json();
  await interaction.editReply(`${data.roast}`);
}javascript
